# ClawPlus 中文讲解教学视频口播稿

适用场景：5-7 分钟、中文普通话旁白、软件实录、不出镜。

建议成片目标：
- 让观众知道 ClawPlus 是什么
- 让观众看懂第一次上手路径
- 让观众记住 3 个能力：能聊天、能接渠道、能做自动化

## 录制前准备

为了让录屏更顺，建议提前准备好这些状态：
- 已完成一次首次配置，避免录制时卡在真实等待过程太久
- Chat 页面里准备 1-2 条成功对话，方便切历史会话
- Models 页面里准备至少 1 个已配置 Provider 和一些 token usage 记录
- Channels 页面里准备 1-2 个已配置渠道，最好有一个在线状态
- Skills 页面里准备几个已启用技能
- Cron 页面里准备 1-2 个任务，其中一个显示下次执行时间
- Agents 页面里准备多 Agent 结构，这样能展示层级关系
- Dashboard 页面里准备可见的运行状态、渠道数和技能数

## 开场 Hook

可选版本 A：

“如果你也想用 AI Agent 帮你处理聊天、接消息、跑自动任务，但又不想天天和命令行打交道，那今天这支视频，你可以把它理解成一个非常直接的上手指南。ClawPlus 做的事情，就是把 OpenClaw 这一类 AI Agent 能力，变成一个普通用户也能快速用起来的桌面软件。”

可选版本 B：

“很多 AI Agent 很强，但真正劝退人的，不是能力不够，而是配置太复杂。ClawPlus 想解决的，就是把原本偏命令行、偏工程化的 Agent 工作流，整理成一套图形化桌面界面。你不用先懂终端，也能先把它跑起来。”

## 时间轴脚本

### 0:00-0:25 开场定位

画面提示：
- 从软件 Logo 或主界面切入
- 快速扫过 Chat、Channels、Cron、Agents 四个页面
- 每个页面停留 2-3 秒，不要细讲

口播稿：

“这就是 ClawPlus。你可以把它理解成一个面向普通用户的 AI Agent 桌面控制台。它不是单纯的聊天壳，而是把聊天、模型配置、消息渠道、自动任务，还有多个 AI 助手的协作，都放进了一个图形界面里。接下来我会用 5 分钟左右，带你看完它最核心的上手路径。”

演示重点：
- 先讲用户收益，不先解释底层架构
- 第一屏就让观众知道它不只是聊天工具

### 0:25-1:20 首次启动与 Setup Wizard

画面提示：
- 进入 Setup 页面
- 依次展示欢迎页、语言选择、环境检查、AI Provider、安装步骤、完成页
- 不必真的等待检查结束，可用已经准备好的状态快速切页

口播稿：

“第一次打开 ClawPlus，会先进入这个 Setup Wizard，也就是首次配置向导。它把原本容易分散在文档、命令行和配置文件里的步骤，收敛成了一条比较顺的流程。比如第一步可以先确认语言和基础设置，接着它会帮你做环境检查。然后最关键的一步，是配置 AI Provider，也就是你要接入哪一家模型服务。你可以把这里理解成先把大脑接上。配置完成之后，向导还会处理默认技能安装，最后再把整个环境带到可用状态。对普通用户来说，这一段最重要的价值就是，第一次上手不用先研究一堆命令再开始用。”

演示重点：
- 光标停留在语言、环境检查结果、Provider 配置框、完成页按钮
- 强调“第一次配置不折腾”

### 1:20-2:10 Chat 主界面

画面提示：
- 回到 Chat 页面
- 点击 `New Chat`
- 输入一条简单问题
- 展示流式回复、历史会话列表、切换会话

口播稿：

“配置完成之后，最自然的入口就是 Chat。这里看起来像聊天窗口，但它更像是你和 AI Agent 协作的工作台。你可以直接新建会话，发出任务，看到回复实时生成。如果左侧已经有历史记录，也可以很快切回之前的上下文继续聊。对于刚开始用的人来说，这一页最重要的不是炫技，而是先把第一轮交互跑通。只要这里能稳定对话，后面像渠道接入、自动任务这些能力，就都是在这个基础上往外扩展。”

演示重点：
- 展示流式输出
- 顺手点一下历史会话，让观众知道有上下文和会话管理

### 2:10-2:50 Models 与 Provider 配置

画面提示：
- 打开 Models 页面
- 展示 Providers 区域
- 向下滚到 token usage history

口播稿：

“如果你想进一步管理模型能力，就可以来到 Models 页面。这里集中处理的是模型供应商，也就是不同 AI 服务的统一配置入口。好处是你不用到处找配置项，常用 Provider 都可以在这里集中管理。下面这部分还能看到 token 使用记录，也就是模型大概用了多少输入、输出和成本。对普通用户来说，这一页的意义很简单，就是把模型接入和使用情况放在一个看得见的位置，不会越用越糊涂。”

演示重点：
- 强调“统一配置”
- token history 只点到为止，不展开算费逻辑

### 2:50-3:25 Channels 消息渠道

画面提示：
- 打开 Channels 页面
- 展示已配置渠道卡片
- 再展示下方支持的渠道列表，如 Telegram、Discord、WhatsApp

口播稿：

“接下来是 Channels，也就是消息渠道。这个功能解决的问题是，AI 不只是待在软件里聊天，它还可以接到外部消息场景里。比如 Telegram、Discord、WhatsApp 这类渠道，你都可以在这里做统一管理。你可以把它理解成，给 AI 接上不同的收件箱和工作入口。这样一来，AI 就不只是等你手动打开软件再提问，而是可以进入你本来就在用的沟通环境里。”

演示重点：
- 先展示已配置渠道，再扫支持渠道
- 不展开每个平台细节，避免信息太杂

### 3:25-4:00 Skills 技能模块

画面提示：
- 切到 Skills 页面
- 展示搜索、启用开关、技能详情抽屉
- 打开一个技能详情页，再关闭

口播稿：

“如果说 Channels 是给 AI 接入口，那 Skills，就是给 AI 加能力。你可以把 Skills 理解成技能模块，像文件处理、代码辅助、工具调用这类能力，都可以通过这里统一安装和管理。普通用户其实不用一开始就装很多，先挑最贴近自己场景的就够了。比如你想让 AI 更偏向内容整理，或者更偏向执行任务，就可以逐步把对应的技能加上去。这样 AI 不是只会聊天，而是会做事。”

演示重点：
- 首次出现 `Skills` 时先说“技能模块”
- 突出“从会聊天到会做事”

### 4:00-4:40 Cron 定时任务

画面提示：
- 进入 Cron 页面
- 打开创建任务弹窗
- 展示任务名称、消息内容、执行时间
- 回到任务列表，看一个已存在任务

口播稿：

“然后是非常实用的一页，Cron，也就是定时任务。它的核心意义很简单，就是让 AI 自动工作，而不是每次都等你手动点开。比如你想让它每天早上 9 点整理日报，或者每隔一段时间检查一次消息，这里都可以直接配置。对很多人来说，真正把 AI 用起来的分界线，不是能不能聊天，而是能不能稳定地替你做重复工作。定时任务就是把这个能力落下来。”

演示重点：
- 用真实例子讲，比如“日报汇总”
- 不讲 cron 表达式原理，只强调“按时间自动执行”

### 4:40-5:30 Agents 多助手协作

画面提示：
- 打开 Agents 页面
- 先展示树状视图，再切换网格视图
- 点击一个 Agent 的设置或 Chat 按钮

口播稿：

“如果你的工作开始变复杂，ClawPlus 还支持多 Agent，也就是多个 AI 助手协作。你不一定要把它理解得很技术化，简单说，就是让不同助手负责不同工作。比如一个助手专门负责写作，一个负责整理资料，另一个负责渠道消息处理。这里还能看到它们之间的层级关系，你可以把它当成一个小型的 AI 团队面板。对于普通用户来说，这一页最值得记住的一点就是，当一个助手不够用的时候，你不需要把所有事情都塞给同一个窗口。”

演示重点：
- 首次出现 `Agent` 时先说“AI 助手”
- 用“不同助手负责不同工作”来代替技术术语

### 5:30-6:10 Dashboard 与 Settings 收尾

画面提示：
- 打开 Dashboard，看状态卡片和 quick actions
- 再切到 Settings，展示主题、语言、代理、迁移备份等区域

口播稿：

“最后看两个收尾页面。Dashboard 更像总览面板，你可以快速看到当前运行状态、连接了多少渠道、启用了多少技能，也能从这里快速跳到常用操作。Settings 则是日常维护页面，比如主题、语言、代理、更新，还有数据迁移和备份这些能力都在这里。也就是说，前面那些是把 AI 用起来，这里则是把它用稳、用顺手。”

演示重点：
- Dashboard 停留在状态卡片和快捷操作
- Settings 只扫重点，不要逐项念菜单

### 6:10-6:30 结尾 CTA

画面提示：
- 回到 Chat 或主导航总览
- 最后停在清晰、干净的一屏

口播稿：

“如果你刚接触这类 AI Agent 工具，我建议的上手顺序非常简单。先配置一个 Provider，先跑通一次聊天；然后根据自己的场景，逐步加上 Channels、Skills 和 Cron。这样你不会一开始就被所有功能淹没，但能很快体会到它真正的价值。如果你想把 AI 从‘偶尔问一问’升级成‘持续帮你做事’，ClawPlus 会是一个很适合开始的入口。”

演示重点：
- 结尾强调下一步动作
- 不要在最后一段再引入新概念

## 术语控制策略

首次出现时，建议统一这样说：

- Agent：先说“AI 助手”，再补一句“在 ClawPlus 里叫 Agent”
- Skills：先说“技能模块”，再补一句“在这里叫 Skills”
- Channels：先说“消息渠道”，再补一句“像 Telegram、Discord 这些入口”
- Cron：先说“定时任务”，再补一句“也就是按时间自动执行”
- Provider：先说“模型供应商”或“模型服务”

避免这样讲：
- 不要一上来连续说 OpenClaw、Gateway、IPC、Transport
- 不要把多 Agent 讲成复杂编排系统
- 不要把 Cron 讲成表达式教学

适合官网或社群：

“从一次成功对话开始，再把渠道、技能和自动任务逐步接上，ClawPlus 的价值会越来越明显。”

## 录制节奏建议

- 全片语速以自然讲解为主，不要像发布会
- 页面切换时，先落光标再说重点
- 每一页只讲一个核心价值，不要把所有功能都念完
- 真正等待时间较长的步骤，建议后期剪掉，保留开始和结果
- 如果某一页数据较空，优先提前准备演示数据，不要靠口头补解释
